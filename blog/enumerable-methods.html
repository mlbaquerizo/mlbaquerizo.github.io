Ruby | Enumerable#map

01-11-16

Classes in Ruby are like blueprints that house various methods and other attributes that can be used on each object in the Class. We can take a step back from that and look at Modules. Modules house various methods that can be used across various Classes. This is just one aspect that makes Ruby such a functional language. It allows us to have methods baked into our own classes by default.

One such Module is the Enumerable Module. We can identify Classes in the Enumerable Module by their ability to use the method each . That is they can be iterated over to yield each successive item in a collection. Many Classes you have probably looked at are Enumerable (contained in the Enumerable Module). For example, Arrays and Hashes are enumerable. We can iterate over them using each.

One of my favorite enumerable methods is the map method. This method can take items from the object that is passed through it, transform those items using a block of code, and return an array of those items transformed. Here's an example:

array = [3, 4, 5, 6]

array.map {|i| i*3}

=> [9, 12, 15, 18]

Pretty cool. With map we can "map" items from a given array to a new array of those items transformed by some code! The possibilities are endless with this method. I should also mention that this method, like many methods in Ruby, has another "twin" method. That would be the collect method. It does the same thing! map and collect could be used interchangably.
